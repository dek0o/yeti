# Предсказание возраста посетителей сайта

**Автор:** Денис Куницын  
**Дата:** 26 марта 2026  

---

### Бизнес-цель
Необходимо понимать возрастную аудиторию сайта на основе цифрового поведения пользователей (время активности, глубина серфинга, предпочитаемые категории сайтов, используемые устройства). 

### Результат
Разработана предиктивная модель, которая классифицирует пользователей на 5 возрастных категорий (<18, 18-25, 26-40, 41-55, 56+). 
Модель успешно распознает внутренние поведенческие паттерны и позволяет таргетировать контент, рекламные предложения и адаптировать интерфейс сайта под конкретную возрастную группу, еще до того, как пользователь явно укажет свой возраст.

### Что сделано:
* Собраны и проанализированы данные о **5913 пользователях** из 6 различных источников (визиты, реклама, устройства, использование облака и т.д.).
* Выявлены ключевые факторы, влияющие на возраст: **основное время активности, предпочтения по категориям сайтов и глубина просмотра**.
* Обучена модель с высокой точностью (качество предсказания превышает заданный бизнесом порог), готовая к интеграции в продакшен.
* Подготовлены все необходимые артефакты для интеграции решения разработчиками.
</details>

### Техническое описание решения
В рамках проекта был реализован плотный и качественный цикл (End-to-End ML Pipeline) разработки модели многоклассовой классификации: от EDA до подготовки артефактов к инференсу.

#### 1. Работа с данным (EDA & Feature Engineering):
* **Агрегация и объединение данных**: Написана функция `build_feature_space`, объединяющая 6 таблиц. Сгенерированы композитные признаки (доля сессий в день, доминирующее время суток, доли категорий сайтов).
* **Качество данных**: Выявлены и обработаны неявные пропуски (замена на константы или категориальные индикаторы), очищены полные дубликаты. Выбросы в поведении (аномальное кол-во сессий) сохранены осознанно как важный сигнал фрода/ботов или уникальных пользователей.
* **Глубокий корреляционный анализ**: Вычислены корреляции **Phi_K** для работы со смешанными типами данных. Идентифицированы и удалены мультиколлинеарные признаки на основе их силы связи с таргетом (`age_category`).

#### 2. ML Pipeline & Modeling:
* **Пайплайн преобразований**: Построен масштабируемый процесс `ColumnTransformer` (кодирование категориальных признаков через `OneHotEncoder`, масштабирование числовых через `StandardScaler`, стабильная импутация).
* **Отбор признаков**: Встроены шаги `VarianceThreshold` (отсев констант) и `SelectPercentile` с функцией `f_classif` для отбора наиболее значимых предикторов.
* **Выбор архитектуры**: Проведен строгий бенчмарк базовых моделей (`DummyClassifier`, `LogisticRegression`, `SVC`) на стратифицированной 5-fold кросс-валидации для контроля распределений. 
* **Оптимизация**: Для лучшей модели `SVC` проведен поиск гиперпараметров (ядро, C, gamma, degree) с помощью `GridSearchCV` с использованием балансировки весов классов (`class_weight='balanced'`).

#### 3. Оценка качества:
* **Метрика**: Оптимизация алгоритма выполнялась по метрике `F1-macro` в условиях умеренного дисбаланса классов.
* **Результаты**: Модель успешно побила жесткий базовый порог `F1-macro >= 0.75` как на CV, так и на отложенном тестовом датасете.
* Для интерпретируемости построена Confusion Matrix (ярко выраженная диагональ отражает уверенное предсказание; просадка на классе 18-25 объяснима его малым объемом в датасете). Извлечены Feature Importances из SelectPercentile.

#### 4. Сохранение модели для продакшена:
* Весь пайплайн и метаданные сохранены в `joblib` словари в `artifacts/`.
* Модуляризован код: выделен python-скрипт `feature_builder.py` для переиспользования логики Feature Engineering в продакшене.
* Окружение зафиксировано в воспроизводимом `requirements.txt`.
</details>

## Структура репозитория
* `Предсказание возраста посетителей сайта.ipynb` — основной Jupyter Notebook с ходом исследований, EDA, обучением и выводами.
* `feature_builder.py` — python-модуль, инкапсулирующий логику преобразования raw-данных в готовые фичи для продакшен-инференса.
* `requirements.txt` — зависимости проекта с точными версиями библиотек.
* `artifacts/` — директория генерируемых артефактов (пайплайн модели `age_model_pipeline.joblib` и метаданные эксперимента `model_metadata.joblib`).
